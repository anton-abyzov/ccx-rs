## Critical Gaps (Blocks Real-World Use)

### 1. Subsystems Built But Never Wired Up

These crates compile and work in isolation but are **never connected to the agent loop**:

| Subsystem | Issue |
|-----------|-------|
| `ccx-sandbox` | `create_sandbox()` called as `let _ = ...` no-op in main.rs; never applied to Bash |
| `ccx-mcp` | MCP client works but never instantiated; no `mcpServers` config read; no tools registered |
| `ccx-memory` | `MemoryStore` defined correctly but never loaded on startup, never injected into system prompt |
| `ccx-compact` | `should_compact()` / `create_summary()` exist but never called from agent loop |

### 2. Agent Tool Missing 7 Parameters
`subagent_type`, `model`, `run_in_background`, `name`, `team_name`, `mode`, `isolation` are all absent.
Makes async/team/permission agent flows non-functional. (The `AgentManager` struct in `ccx-core` exists but isn't wired either.)

### 3. FileRead: No Image/PDF Support
TS returns base64-encoded images (JPEG/PNG/GIF/WebP) and PDFs with metadata. Rust returns `"binary file"` message. Breaks all vision and document workflows.

### 4. Default Permission Mode is `bypass` (not `default`)
TS ships with `default` mode (prompts for unrecognized tools). ccx defaults to `bypass` — no permission checks run unless explicitly set. Opposite of safe-by-default.

### 5. No Prompt Caching Header
`anthropic-beta: prompt-caching-2024-07-31` is never sent. The system prompt is a plain `Option<String>` instead of a structured array with `cache_control` blocks. Prompt caching never activates.

### 6. Sequential Tool Execution
TS executes all tools in a turn in parallel (`Promise.all`). Rust executes sequentially. Significant performance gap for multi-tool turns.

## API Type Gaps (ccx-api)

- `ContentBlock` has no `Image` variant — image content in tool results unsupported
- `ToolResult` accepts only `String` content (not structured array with images)
- `MessageRequest` missing: `top_p`, `top_k`, `metadata`, `stop_sequences`, `betas`, `thinking`
- `StopReason::PauseRun` not modeled (needed for plan mode / human-in-the-loop)

## Rust Extensions (Beyond TS Spec — Good)

- `FileEdit`: atomic write via temp-file + rename
- `Glob`: sort by modification time
- `Grep`: 30-second internal timeout
- `Bash`: stdin(null) to prevent interactive hangs; key env var passthrough
- `WebFetch`: `max_size` parameter
- `WebSearch`: DuckDuckGo fallback when no API key
- `TeamCreate`: novel team management tool (ccx-rs specific)
- OAuth/keychain token reuse for Claude.ai subscribers

## Recommended Fix Order

1. **Wire the unconnected subsystems** — sandbox → Bash tool; MCP → ToolRegistry; memory → startup + system prompt; compact → agent loop post-turn check
2. **Prompt caching** — add `anthropic-beta: prompt-caching-2024-07-31` header; convert system to structured array with `cache_control`
3. **Default permission mode** — change default from `bypass` to `default`
4. **FileRead image/PDF support** — detect MIME type, return base64 block
5. **Agent tool parameters** — add `subagent_type`, `model`, `mode`, `name`, `team_name`, `run_in_background`
6. **Parallel tool execution** — `tokio::join_all` for tool calls in a single turn
7. **Extended thinking request** — add `thinking` field to `MessageRequest` + beta header
8. **Project-level settings** — load `.claude/settings.json` from cwd and merge into cascade
9. **WebFetch `prompt` field** — add LLM post-processing pass after fetch
10. **NotebookEdit** — switch to `cell_id` string addressing + add `insert`/`delete` edit modes
